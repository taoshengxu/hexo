3. 查找PERL路径

I know this is an old thread, but I’ll post my answer here in case it helps others. I was having the same problem. I think perl is failing to find the module. First, if you have multiple perl installations, make sure you know which version you’re using with

which perl.

perl -E 'say for @INC' or perl -e "print for @INC"to identify where perl is looking for the modules.

Alternatively, since I work on a big server and I don’t have root privileges, I just installed the module in my home directory (~/perl5/lib/perl5/x86_64-linux-thread-multi/Text/Soundex.pm). PERL5LIB=~/perl5/lib/perl5/x86_64-linux-thread-multi/:$PERL5LIB to add my own installation to the top of my perl lib path.

5. 非root用户安装perl模块

CentOS7系统默认设置下，每当用户登录时，会自动在其家目录下生成名为perl5的文件夹。该文件夹是用于安装用户自己的perl模块，以利于用户使用一些生物信息学软件对perl模块的需求。这点对生物信息学软件非常重要。如果没有用户perl文件夹，会导致没有root权限的人无法安装perl模块。
